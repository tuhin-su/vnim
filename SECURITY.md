# Security Policy

## Supported Versions

We actively support and patch security vulnerabilities in the following versions of VNIM:

| Version | Supported          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

## Reporting a Vulnerability

We take the security of VNIM seriously. If you discover a security vulnerability, please do **not** open a public issue. Instead, report it responsibly:

1. Send an email or reach out to the project maintainers (via **tuhin-su** on GitHub).
2. Please include detailed steps to reproduce the vulnerability, including configuration payloads, logs, or proof-of-concept code.
3. We will acknowledge receipt of your report within 48 hours and work with you to analyze and resolve the issue.
4. Once a fix has been prepared and tested, we will publish a security advisory and release a patched version.

Thank you for helping keep VNIM secure!
